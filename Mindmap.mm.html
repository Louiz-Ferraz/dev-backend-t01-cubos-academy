    • + - Métodos de Arrays II
      • every(callback)
        • testa todos os elementos do array
        • se passam pelo teste implementado na função callback
        • callback é passada como argumento
        • retorna true ou false
        • ex.:
          • fruits.every((currentFruit) => {
                return currentFruit !== "banana";
            })
      • some(callback)
        • testa se pelo menos um dos elementos do array
        • se passa pelo teste da função callback
        • callback é passada como argumento
        • retorna true ou false
        • ex.:
          • names.some((name) => {
                return name === "Luiz";
            })
      • find(callback)
        • retorna o primeiro elemento do array que corresponde à condição da função callback
        • caso não encontre, retorna undefined
        • se o array for de objetos, ele retorna o primeiro objeto completo encontrado
        • ex.:
          • users.find((user) => {
                return user.name === "Al";
            })
      • findIndex(callback)
        • retorna o índice do pirmeiro elemento do array
        • que corresponda à condição da função callback
        • caso contrário, retorna -1
        • ex.:
          • cars.findIndex((car) => {
                return car.name === "Mustang";
            })
      • filter(callback)
        • cria um novo array com todos os elementos
        • elementos que correspondem à condição do callback
        • não altera array original
        • caso não encontre, retorna array vazio
        • ex.:
          • numbers.filter((number) =>  {
                return number === 4;

    • + - Autenticação e Criptografia
      • Criptografia
        • Comunicação segura
        • Estabelecer confiança entre duas partes
          • sem precisar confiar em intermediários
        • Princípios
          • Confidencialidade
          • Integridade
            • Não pode ser alterada
          • Autenticação
            • Possível verificar a origem
          • Não-repúdio
            • Posso provar para outros que mensagem é autêntica e veio da fonte verdadeira
      • Teoria da informação
        • uma mesma informação pode ser representada de várias formas
          • 137 ou "centro e trinta e sete"...
        • a informação é abstrata, a representação é concreta
        • na criptografia, trabalhamos com representação
      • Ofuscação
        • Procedimento secreto para transformar dados e torná-los incompreensíveis
        • junto a procedimento secreto para recuperar informação original
        • É a origem da criptografia
        • Não traz segurança
          • Uma vez que o procedimento é sabido, todo histórico já era
      • Esteganografia
        • Ocultar informação de forma que não seja percebida
        • Muito comum em fotos
          • colocar informação em uma foto, pequena, disfarçada
        • Não traz segurança
      • Criptografia simétrica
        • uma cifra simétrica é um par de procedimentos (E e D)
          • em que é possível encriptar mensagem M
          • com uma chave K
          • gerando um texto cifrado C indistinguível de aleatório
          • e é possível reverter apenas com a chave K
        • ou seja
          • E(M, K) = C
            • encriptografia
          • D(C, K) = M
            • desencriptografia
        • Propriedades
          • mesmo com M e C, não dá pra deduzir K
          • a mesma chave K pode ser utilizada em várias mensagens
          • os procedimentos E e D são publicamente conhecidos
            • são alvo de muita pesquisa
          • se vc perder K, é impossível recuperar M
          • exemplo procedimento
            • AES-CBC
      • Hash
        • Função de sentido único
          • one way function
          • procedimento fácil de ser realizado
          • dado o resultado, é muito difícil reverter
        • Hash criptográfico
          • para uma mensagem M, é possível calcular H(M) rapidamente
          • Dado H(X), não é possível achar X
          • H(X) é indistinguível de dados aleatórios
          • Não dá pra encontrar X e Y tais que H(X) = H(Y)
          • Exemplo:
            • SHA-256
          • Aplicação
            • verificar integridade ou posse
      • Assinatura digital
        • um procedimento de assinar e um de verificar
        • a assinatura é calculada a partir do hash da mensagem e uma chave secreta
          • essa assinatura não revela nada da mensagem original
        • da posse da mensagem original, assinatura e da chave
          • é possível verificar se a assinatura é legítima
        • qualquer modificação na mensagem invalida a assinatura
        • Exemplo
          • HMAC
        • Assinatura digital assimétrica
          • usa uma chave privada pra assinar
            • uma chave pública pra verificar
              • a chave pública é divulgada abertamente
          • apenas quem tem a chave privada pode assinar
          • qualquer um consegue validar a assinatura
          • muito comum em documentos
